Build `AudioRecorderView.swift` in `Sources/FeatureAudioRecorder/`. The view should be a compact, focused panel following the same visual style as BPMTapperView.

Key UI elements (in order from top):

1. **Header section** (VStack, alignment: .leading, spacing: 8):
   - Label with systemImage "waveform.circle" and title "Audio Recorder"
   - Status text showing current `recordingState`

2. **Filename section** (compact text field or display):
   - If `filenameOverride.isEmpty`: show "Recording YYYY-MM-DD HH-mm-ss.wav" in secondary color
   - If override set: show the override value
   - No edit during recording

3. **Meter section** (ZStack with RoundedRectangle background):
   - Two bars (left/right channel) or single combined bar showing `currentLevel.peak`
   - Peak meter fills from 0..1 horizontally
   - Color: green when < 0.7, yellow 0.7..0.9, red > 0.9
   - Hide when idle (opacity 0)
   - Update at ~30Hz via the async level stream

4. **Time section** (monospaced, large font):
   - Display elapsed time as `HH:MM:SS` using `elapsedTime`
   - Updates live during recording
   - Show 00:00:00 when idle

5. **Control section** (HStack with buttons):
   - Start button (disabled when already recording):
     - Label: "Start Recording" or "Record"
     - Style: `.borderedProminent`
     - Color: .red when recording available, .accentColor otherwise
   - Stop button (only visible during recording):
     - Label: "Stop"
     - Style: `.bordered`
     - Color: .red
   - Spacebar triggers Start when view is focused (use `@FocusState` and `.onKeyPress(.space)` like BPMTapperView)
   - When recording: Start button changes to Stop button (use conditional rendering)
   - Recording indicator: pulsing red circle when recording (animated with `.animation(.easeInOut(duration: 0.8).repeatForever(autoreverses: true))`)

6. **Settings section** (collapsible or always visible):
   - Max duration picker: Picker with options [5 min, 10 min, 15 min, 30 min, 60 min, Unlimited] bound to `maxDurationMinutes`
   - Show current setting in a compact row

7. **Error section** (VStack with error display):
   - Only visible when `recordingState` is `.error(...)`
   - Shows the error message from `RecorderError` as a localizedDescription
   - "Open System Settings" button for `.permissionDenied(needsSettings: true)` — opens System Settings pane
   - "Try Again" button for other errors to reset to idle
   - "Retry" button for API errors

8. **Permission guidance section** (only visible when `recordingState` is `.permissionNeeded`):
   - Explains why permission is needed
   - "Grant Permission" button that calls `requestPermission()`
   - After permission granted, auto-dismiss and update state

9. **Incompatible macOS section** (only visible when `recordingState` is `.incompatibleMacOS`):
   - Shows current macOS version and minimum required (14.2)
   - Explains what to do (use external audio interface or upgrade macOS)

```swift
struct AudioRecorderView: View {
    let context: ToolContext
    @StateObject private var viewModel: AudioRecorderViewModel
    @FocusState private var recorderFocused: Bool

    public init(context: ToolContext) {
        self.context = context
        // Initialize viewModel with dependencies from context
    }

    var body: some View {
        ScrollView {
            VStack(alignment: .leading, spacing: 24) {
                header
                filenameDisplay
                meterSection
                timeDisplay
                controlSection
                settingsSection
                errorSection
                permissionSection
                incompatibleSection
            }
            .padding(.horizontal, 24)
            .padding(.bottom, 24)
            .padding(.top, 32)
            .frame(maxWidth: 640, alignment: .topLeading)
        }
        .frame(maxWidth: .infinity, maxHeight: .infinity, alignment: .topLeading)
        .background(Color(nsColor: .windowBackgroundColor))
        .focusable()
        .focused($recorderFocused)
        .onAppear { recorderFocused = true }
        .onKeyPress(.space) {
            if viewModel.isRecording { viewModel.stopRecording() }
            else { Task { await viewModel.startRecording() } }
            return .handled
        }
    }
}
```

Note: `viewModel.isRecording` is derived from `recordingState == .recording`.
