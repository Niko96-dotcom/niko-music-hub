# Phase 03 - UI Design Contract

> Visual and interaction contract for Phase 3: Cubase-Ready WAV Conversion. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 3 adds the first file-producing audio tool to Outside Cubase Hub: a compact native converter that accepts M4A, MP3, WAV, AIFF, and FLAC sources, converts them into verified WAV PCM output using the current project audio preset, and records successful outputs in the shared output inbox for reveal and drag-out into Cubase.

The UI must make three things obvious without becoming a full audio editor: which files are queued, what WAV spec will be produced, and which outputs are verified and ready to hand off. It must also keep helper-tool fallback failures recoverable per file, so one FLAC needing FFmpeg does not obscure native-convertible files in the same batch.

Do not add waveform editing, trim/fade controls, loudness/BPM/key analysis, recursive folder scanning, downloader behavior, internal recording, Cubase project watching, or naming-template editing in this phase.

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual native macOS SwiftUI contract |
| Preset | Not applicable |
| Component library | SwiftUI system controls with AppKit interop for file import, Finder reveal, and drag-out file URLs |
| Icon library | SF Symbols |
| Font | macOS system font, SF Pro Text/Display via SwiftUI `.system` |

### Component Contract

| Component | Contract |
|-----------|----------|
| Converter registration | Register as a normal `ToolFeature`; sidebar short label is `WAV Converter` with an SF Symbol that reads as audio conversion or waveform output, such as `waveform.path` if available or `waveform` as fallback. Capabilities must include `.producesFiles` and `.runsJobs`. |
| Tool header | Compact title row inside the active tool detail area. Show `WAV Converter` and one short status line summarizing selected count, conversion state, or the last recoverable problem. |
| Source intake | Primary intake surface supports drag-in and a native choose-file action for M4A, MP3, WAV, AIFF, and FLAC. Dropped folders scan top-level supported audio files only. The surface must not imply recursive sample-pack ingestion. |
| Preset summary | A compact spec strip shows the current output preset before conversion: sample rate, bit depth, and channel handling. The default visible spec is `44.1 kHz - 24-bit - Preserve mono/stereo`. |
| Preset edit affordance | If editable in this phase, use native controls near the spec strip for sample rate, bit depth, and channel handling. Keep edits compact and clearly tied to the shared project preset. |
| Batch list | Dense per-file rows with source filename, detected type when known, planned output name, per-file status, progress, and the converter path used after completion (`Native` or `FFmpeg`). |
| Primary actions | Text buttons for `Add Audio Files`, `Convert to WAV`, and `Stop After Current File`. Use bordered prominent styling only for `Convert to WAV` when at least one convertible file is queued. |
| Per-file recovery | Failed rows keep the source visible, show a short cause, and expose a specific recovery action when one exists, such as `Choose FFmpeg` for fallback-required files. |
| Results handoff | Successful rows expose `Reveal in Finder` and drag-out behavior for the verified WAV. The shared output inbox also lists the same verified outputs with source file, specs, and converter metadata. |
| Output inbox | Remains the shared secondary inspector from the shell. Converter results may appear in the active tool, but the inbox is still the durable handoff surface. |

## Layout And Interaction

| Area | Contract |
|------|----------|
| Detail area | Keep the current app shell constraints: active tool minimum width 540px inside the split view and a 320px output inbox inspector. Converter content should remain usable at the 1220px minimum window width established by the shell. |
| Content width | Constrain the main converter work area to an ideal width of 560-720px and align it top-leading. Do not stretch file rows across ultra-wide desktop space. |
| Intake surface | Place the drag/choose surface first. It is the primary visual anchor when no batch is running. Keep its height stable from empty through drag-over states. |
| Preset strip | Place directly below the intake surface so users see the WAV target before pressing `Convert to WAV`. Use one line where possible and wrap into labeled chips only when the window is narrow. |
| Batch list | Place queued and completed rows below the preset strip. The list must support mixed states in one batch: queued, converting, verified, failed, unsupported, and waiting for stop. |
| Progress | Show overall batch progress in the header or above the list, and per-file progress in each active row. Unknown progress may use a native indeterminate indicator plus a specific message. |
| Actions | Keep `Add Audio Files`, `Convert to WAV`, and `Stop After Current File` near the intake and preset summary. Disable actions that do not have a valid target. |
| Cancellation | Stopping a batch cancels remaining files while allowing the current file to finish safely. The UI copy must never suggest an already verified WAV will be deleted. |
| Filename handling | Show the planned output filename before conversion. If a counter suffix is added to avoid overwrite, reflect the final filename in the verified row and output inbox metadata. |
| Drag-out | Dragging a verified result row or output inbox item must provide the WAV file URL. Unverified, failed, pending, or missing rows must not be draggable as ready outputs. |
| Keyboard | Native tab order must reach intake, preset controls, primary actions, batch rows, per-file actions, result reveal, and output inbox actions. |

### Visual Hierarchy

The primary focal point is the source intake surface when idle and the active converting row when a batch is running. The preset strip is the second focal point because it answers "what WAV will Cubase receive?" before the user commits. Verified result rows and the output inbox are the handoff layer.

The eye should move in this order:

1. Source intake surface or active converting row.
2. Preset summary and output naming preview.
3. `Convert to WAV` or current batch progress.
4. Per-file status and recovery actions.
5. Verified results with reveal and drag-out.
6. Shared output inbox inspector.

Icon-only controls must have accessibility labels and tooltips. Use text labels for converter commands; reserve icon-only buttons only for familiar secondary actions such as row reveal or remove-from-queue when the row already has a visible label.

## Spacing Scale

Declared layout values. All custom spacing tokens are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/text gaps, metadata separators, status dot gaps |
| sm | 8px | Compact row gaps, file-type badges, helper message spacing |
| md | 16px | Default group spacing, action rows, batch-row inner padding |
| lg | 24px | Tool header spacing, intake section padding, preset section padding |
| xl | 32px | Gap between intake/preset area and batch results |
| 2xl | 48px | Empty-state vertical rhythm inside the intake area |
| 3xl | 64px | Rare wide-window breathing room only |

Exceptions: none for custom layout spacing. Native SwiftUI controls may keep platform intrinsic heights, but custom padding and gaps must use the tokens above.

## Typography

Use exactly four text sizes and exactly two weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Caption / metadata | 12px | Regular 400 | 1.35 |
| Body | 13px | Regular 400 | 1.45 |
| Section heading / tool title | 16px | Semibold 600 | 1.25 |
| Screen title / primary file status | 22px | Semibold 600 | 1.20 |

Letter spacing: 0. Do not scale font sizes with viewport width. Use monospaced digits only for sample rate, bit depth, channel count, file counts, and progress percentages. Long filenames and paths may truncate in the middle or tail with native tooltips/accessibility values preserving the full text.

## Color

Reuse the Phase 1 and Phase 2 native palette so the converter feels like part of the same production bench. Prefer native semantic colors in SwiftUI, backed by these fixed reference values for planning and review.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F7F8FA | Window background and main converter detail surface |
| Secondary (30%) | #E7EAEE | Sidebar, output inbox, intake surface fill, batch rows, preset strip, dividers |
| Accent (10%) | #0B6E69 | `Convert to WAV`, drag-over border, active sidebar indicator, keyboard focus ring, running progress fill |
| Destructive | #C7362F | Stop-batch confirmation emphasis, failed row status, destructive row removal only |

Accent reserved for: `Convert to WAV`, drag-over border, active sidebar indicator, keyboard focus ring, and running progress fill. Do not use the accent for every button, every file badge, every icon, or decorative backgrounds.

State colors:

| State | Value | Usage |
|-------|-------|-------|
| Verified | #2F7D32 | Verified WAV status and inline completion text only |
| Failed | #C7362F | Failed conversion state and destructive emphasis |
| Warning | #8A5A00 | Missing FFmpeg, unsupported file, or overwrite-counter notice |
| Pending / muted | #6B7280 | Queued files, inactive metadata, skipped remaining files |
| Running | #0B6E69 | Active conversion progress and drag-over/focus emphasis |

## Copywriting Contract

Use terse, concrete labels. The converter should sound like a working studio utility, not a tutorial or marketing page.

| Element | Copy |
|---------|------|
| Primary CTA | Convert to WAV |
| Add files action | Add Audio Files |
| Stop action | Stop After Current File |
| Preset edit action | Edit WAV Preset |
| Missing helper action | Choose FFmpeg |
| Reveal action | Reveal in Finder |
| Row drag label | Drag WAV to Cubase |
| Empty state heading | Drop audio files to convert |
| Empty state body | Add M4A, MP3, WAV, AIFF, or FLAC files. Outputs use the current WAV preset and appear here after verification. |
| Drag-over state | Release to add supported audio |
| Queued state | Ready for WAV conversion |
| Running state | Converting to Cubase-ready WAV |
| Verified state | Verified WAV ready |
| Unsupported file state | This file type is not supported in Phase 3. Add M4A, MP3, WAV, AIFF, or FLAC instead. |
| Missing FFmpeg state | FFmpeg is required for this file. Choose FFmpeg, then convert this file again. |
| Verification failure state | WAV verification failed. The source file was left untouched; check the output preset and try again. |
| Output exists notice | Filename already exists. A numbered WAV name will be used. |
| Error state | Could not convert this file. Keep the source selected, review the row message, then try Convert to WAV again. |
| Destructive confirmation | Stop After Current File: Finish the current conversion, then skip the remaining queued files? Verified WAVs stay in the output folder. |

Avoid generic one-word command labels such as `Submit`, `OK`, `Cancel`, `Save`, `Start`, or `Stop` in the converter surface. Use action-specific labels from the table.

## Empty, Loading, And Error States

| State | Contract |
|-------|----------|
| No files selected | Show the empty-state copy inside the intake surface, with `Add Audio Files` nearby and `Convert to WAV` disabled. |
| Dragging supported files | Show drag-over copy and accent border. Do not change layout size during drag-over. |
| Dragging unsupported files | Keep the source intact and show unsupported-file copy inline after drop; do not block other supported files in the same drop. |
| Folder dropped | Scan top-level supported audio files only and show a compact notice if subfolders were ignored. |
| Files queued | Show each source filename, file type when known, planned output name, and `Ready for WAV conversion`. |
| Batch running | Show overall progress, active row progress, converter path when known, and `Stop After Current File` when cancellation is available. |
| Native conversion failed with fallback available | Keep the row active, show that FFmpeg fallback is being tried, and record the final converter path after success or failure. |
| FFmpeg missing | Fail only the affected row with missing-helper copy and `Choose FFmpeg`; continue other native-convertible files. |
| Verification failed | Mark only that row failed, keep the source untouched, exclude the output from drag-out and the ready output inbox state. |
| Batch stopped | Mark remaining queued files as skipped, keep completed verified rows available, and leave sources in the list for retry. |
| Verified output | Show verified specs, final filename, converter used, `Reveal in Finder`, and drag-out affordance. Add the item to the output inbox. |
| Output file missing later | In the output inbox, mark the item missing and hide drag-out/readiness affordances. Offer `Reveal in Finder` only when the parent folder exists. |

## Accessibility And Native Behavior

| Requirement | Contract |
|-------------|----------|
| VoiceOver | Intake surface, preset strip, primary actions, per-file status, progress, reveal actions, and drag-ready rows must have accessibility labels. Verified rows should announce filename, sample rate, bit depth, channel handling, and converter path. |
| Focus | Initial focus should land on the intake surface or `Add Audio Files`. Focus rings use the accent color only. |
| Keyboard | Native tab order must support adding files, editing preset controls, starting conversion, stopping after current file, row recovery actions, and reveal actions without pointer-only interactions. |
| Drag and drop | Drag-in accepts supported audio file URLs and top-level folder contents. Drag-out is available only for verified WAV file URLs. |
| Contrast | Text, status colors, accent progress, warning, and destructive color must meet WCAG AA contrast against their surfaces. |
| Motion | Use native progress changes and subtle row state transitions only. Avoid decorative animation while conversion is running. |
| File safety | Failed, unverified, pending, or skipped rows must never expose the same drag-ready affordance as verified rows. |
