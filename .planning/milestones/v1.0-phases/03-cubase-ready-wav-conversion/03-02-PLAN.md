<threat_model>
Trust boundaries:
- FFmpeg path is local configuration and may be missing, stale, or point to an unexpected executable.
- Source file paths and output paths are untrusted strings crossing into a helper process.
- Process stdout/stderr are external data and should become typed diagnostics, not raw UI control flow.

STRIDE register:
- threat_id: T-03-PROCESS
  category: Elevation of Privilege
  component: FFmpegAudioConverter
  disposition: mitigate
  severity: high
  mitigation_plan: Launch FFmpeg with `Process.executableURL` and `[String]` arguments only; source-scan for `/bin/sh` and `sh -c`; unit-test exact argv arrays.
- threat_id: T-03-HELPER
  category: Denial of Service
  component: FFmpegHealthChecker
  disposition: mitigate
  severity: medium
  mitigation_plan: Represent missing helper as a typed recoverable error for the affected file instead of failing the whole batch.
- threat_id: T-03-VERIFY-FALLBACK
  category: Tampering
  component: FFmpegAudioConverter
  disposition: mitigate
  severity: high
  mitigation_plan: Run `WAVOutputVerifier` after FFmpeg exits successfully and remove temp output on mismatch.
</threat_model>
