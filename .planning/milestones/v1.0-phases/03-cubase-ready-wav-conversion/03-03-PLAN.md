<threat_model>
Trust boundaries:
- Dragged/chosen file URLs are local user input and may include unsupported files, folders, missing files, or paths outside the output folder.
- Batch conversion uses async background work and must keep UI state consistent with job progress.
- The output inbox is the durable handoff surface and must only receive verified WAVs.

STRIDE register:
- threat_id: T-03-INTAKE
  category: Tampering
  component: AudioFileIntakeScanner
  disposition: mitigate
  severity: medium
  mitigation_plan: Accept only explicit supported extensions/types and scan dropped folders one level deep without recursion.
- threat_id: T-03-BATCH
  category: Denial of Service
  component: BatchAudioConversionUseCase
  disposition: mitigate
  severity: medium
  mitigation_plan: Process files sequentially, update progress per file, and stop only between files when requested.
- threat_id: T-03-INBOX
  category: Tampering
  component: BatchAudioConversionUseCase
  disposition: mitigate
  severity: high
  mitigation_plan: Add `OutputInboxItem(status: .available)` only after converter result includes a verified output spec.
</threat_model>
