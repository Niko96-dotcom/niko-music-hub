<threat_model>
Trust boundaries:
- Saved BPM history is local user data stored in UserDefaults.
- Clipboard writes cross from the app into the user's system pasteboard.
- Clear-all history is destructive for saved BPM rows but must not affect the current tap run.

STRIDE register:
- threat_id: T-02-LOCALDATA
  category: Tampering
  component: UserDefaultsBPMHistoryStore
  disposition: mitigate
  severity: medium
  mitigation_plan: Encode/decode typed `BPMHistoryEntry` values, expose append/list/clear operations, and cover persistence with isolated UserDefaults suites.
- threat_id: T-02-CLIPBOARD
  category: Information Disclosure
  component: BPMClipboardWriter
  disposition: mitigate
  severity: low
  mitigation_plan: Write only the displayed plain BPM number string and test that no mode/timestamp/unit text is copied.
- threat_id: T-02-CLEAR
  category: Repudiation
  component: Clear History action
  disposition: mitigate
  severity: low
  mitigation_plan: Require explicit confirmation copy before clearing all saved rows and leave the current tap run intact.
</threat_model>
