<objective>
Create the shared settings, output inbox, and job primitives that all later file-producing tools will use.

Purpose: Future conversion, recording, and downloader phases need durable settings, a Cubase-ready output handoff model, and one shared job state machine. This plan turns those foundations into tested AppCore services and lightly wires them into the Phase 1 shell without implementing any later tool behavior.

Output: Persistent app settings, audio preset defaults, helper path settings, JSON output inbox metadata store, file action contracts, shared job model/runner, UI surfaces for output folder/inbox/job state, and XCTest coverage for persistence and state transitions.
</objective>

<threat_model>
Trust boundaries:
- Settings and output metadata are local user data and must remain local to the Mac.
- Output folder and helper tool paths are user-controlled file URLs and may be stale, missing, or unwritable.
- Jobs run local async work only in Phase 1; external helper execution remains out of scope.

STRIDE register:
- threat_id: T-01-FILE
  category: Tampering
  component: UserDefaultsSettingsStore / JSONOutputInboxStore
  disposition: mitigate
  severity: medium
  mitigation_plan: Persist output folder through a URL/bookmark-shaped abstraction, validate local paths before reveal, and write output inbox JSON atomically.
- threat_id: T-01-JOB
  category: Denial of Service
  component: JobRunner
  disposition: mitigate
  severity: medium
  mitigation_plan: Use explicit queued/running/completed/failed/canceled transitions, cancellation hooks, and no unbounded detached loops.
- threat_id: T-01-SETTINGS
  category: Information Disclosure
  component: HelperToolSettings
  disposition: mitigate
  severity: low
  mitigation_plan: Store optional local helper paths only; do not log full helper paths in normal UI rows.
</threat_model>
