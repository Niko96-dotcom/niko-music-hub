Create `DownloaderUseCase.swift`:
- `private let downloader: DownloadRunning`, `private let healthChecker: YtDlpHealthChecker`, `private let jobRunner: JobRunner`, `private let settingsStore: SettingsStore`
- `func simulateAndEnqueue(url: URL, options: DownloadJobOptions) async throws -> Job`:
  1. Check yt-dlp availability via `healthChecker.availability(settings: ...)`
  2. If `.missing` or `.unusable`, throw `DownloadUseCaseError.ytDlpUnavailable`
  3. Build `DownloadRequest` and call `downloader.download(...)` with a short timeout (~30s) to validate URL
  4. If simulation fails (nonzero exit), throw `DownloadUseCaseError.unsupportedURL` with yt-dlp stderr
  5. Enqueue job via `jobRunner.enqueue(title: "Download: \(url.lastPathComponent)")` with the actual download operation
  6. Return the Job
- Retry logic with exponential backoff (2s, 4s, 8s) for transient failures in the enqueued job operation
- On final failure, clean up partial downloads and mark job failed

```swift
@MainActor
public final class DownloaderViewModel: ObservableObject {
    @Published public var urlText: String = ""
    @Published public var detectedFileName: String?
    @Published public var downloadState: DownloadState = .idle
    @Published public var statusMessage: String?
    @Published public var errorMessage: String?

    public enum DownloadState: Equatable {
        case idle
        case checkingURL
        case readyToDownload
        case downloading
        case completed
        case failed(String)
    }
}
```

State machine:
- `idle` — empty URL field
- `checkingURL` — user pasted URL, running simulate
- `readyToDownload` — simulate succeeded, file name detected, Download button enabled
- `downloading` — job running, progress visible
- `completed` — success, output in inbox
- `failed(String)` — error with actionable message

Methods:
- `urlTextDidChange()` — debounce 500ms, then run simulate check if valid URL
- `startDownload()` — calls `useCase.simulateAndEnqueue(...)`, transitions to `.downloading`
- `clearInput()` — resets to idle
- `observeJob(id: Job.ID)` — subscribes to job runner for progress updates

When job completes successfully: call `outputInboxStore.addItem(...)` with `dlSourceURL` metadata and transition to `.completed`.

When job fails: transition to `.failed(message)`.
