# Requirements: Niko Music Hub — v1.4 (archived)

**Defined:** 2026-06-11  
**Shipped:** 2026-06-11  
**Milestone:** v1.4 Downloader Reliability

## v1.4 Requirements (21/21 complete)

### Command Truth

- [x] **CMD-01**: User sees real downloader progress from the app's actual `yt-dlp` command.
- [x] **CMD-02**: Long valid downloads are not killed solely because they exceed 90 seconds.
- [x] **CMD-03**: Stalled downloads fail with a clear stall/timeout message after no meaningful output or progress.
- [x] **CMD-04**: Downloader preflight validates the selected format path and prevents accidental playlist expansion.
- [x] **CMD-05**: Downloader output collection survives split UTF-8 chunks and still finds final output files.

### Helper Health

- [x] **HLTH-01**: User can see when `yt-dlp` is missing, unusable, available, or outdated.
- [x] **HLTH-02**: Outdated `yt-dlp` guidance tells the user how to update through the app's existing helper flow.
- [x] **HLTH-03**: Retry behavior matches real transient failures, including "timed out" wording and common network failures.
- [x] **HLTH-04**: Downloader jobs use human-readable titles instead of generic URL path fragments like `watch`.

### Output Contract

- [x] **OUT-01**: Completed downloader jobs pass output file URLs as structured data to inbox ingestion.
- [x] **OUT-02**: Downloader inbox ingestion no longer depends on synthetic `[download] Destination:` log lines.
- [x] **OUT-03**: Diagnostic logs remain useful for debugging without becoming the source of truth for output files.

### Media Handoff

- [x] **HAND-01**: User can reveal safe completed downloader media files in Finder.
- [x] **HAND-02**: User can open safe completed downloader media files from the output inbox.
- [x] **HAND-03**: User can drag safe completed downloader media files from the output inbox to Finder/Cubase-compatible targets.
- [x] **HAND-04**: Output inbox shows appropriate icons/status for common downloader media types such as MP3, M4A, MP4, and WEBM.
- [x] **HAND-05**: WAV verification remains strict for converter/recorder outputs while downloader media uses an explicit safe media allowlist.

### Real UAT

- [x] **UAT-01**: v1.4 includes deterministic tests for progress markers, stall handling, helper health, structured output handoff, and media handoff.
- [x] **UAT-02**: v1.4 includes opt-in/live downloader verification that exercises real `yt-dlp` behavior beyond the previous 18-second happy path.
- [x] **UAT-03**: v1.4 verification proves helper-path behavior from app-like stripped environments still works.
- [x] **UAT-04**: Milestone close includes documented evidence for downloader success, failure, progress, and media handoff flows.
