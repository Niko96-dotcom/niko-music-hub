# Milestone v1.0 Requirements Archive

**Milestone:** Outside Cubase Hub v1.0 MVP
**Archived:** 2026-05-11
**Total v1 Requirements:** 32

## Requirements Status

### Foundation — All Validated ✅

- [x] **FND-01**: User can open a native macOS app shell that lists registered production tools in a sidebar or equivalent primary navigation. — Phase 1 validated
- [x] **FND-02**: Developer can add a new tool by registering a feature module without editing existing feature internals. — Phase 1 validated
- [x] **FND-03**: User can configure a shared output folder/inbox used by all file-producing tools. — Phase 1 validated
- [x] **FND-04**: User can see shared job states for queued, running, completed, failed, and canceled work. — Phase 1 validated
- [x] **FND-05**: User settings persist locally, including audio preset defaults and helper tool paths. — Phase 1 validated

### BPM Tapper — All Validated ✅

- [x] **BPM-01**: User can tap tempo by clicking and by using the keyboard. — Phase 2 validated
- [x] **BPM-02**: User can see a live BPM estimate that stabilizes over recent taps. — Phase 2 validated
- [x] **BPM-03**: User can reset taps and apply half-time or double-time adjustments. — Phase 2 validated
- [x] **BPM-04**: User can copy or save the current BPM result to a local history. — Phase 2 validated
- [x] **BPM-05**: User can use the BPM tapper offline without opening an external website. — Phase 2 validated

### Conversion — All Validated ✅

- [x] **CONV-01**: User can drag in or choose M4A and common audio files for conversion. — Phase 3 validated
- [x] **CONV-02**: User can convert selected files to WAV PCM using the current project audio preset. — Phase 3 validated
- [x] **CONV-03**: User can choose or edit sample rate, bit depth, and channel handling for WAV output. — Phase 3 validated
- [x] **CONV-04**: User can run batch conversions with per-file progress, success, and error states. — Phase 3 validated
- [x] **CONV-05**: App verifies output WAV metadata and records the result in the shared output inbox. — Phase 3 validated

### Recording — Partial ⚠️

- [ ] **REC-01**: App checks macOS compatibility and explains required system audio recording permission before capture. — Phase 4 partial (synthetic audio, needs real system verification)
- [ ] **REC-02**: User can start and stop internal Mac/system audio recording. — Phase 4 partial (UI implemented, real interaction unverified)
- [ ] **REC-03**: Recording writes a playable WAV PCM file to the shared output inbox. — Phase 4 partial (WAV writer uses synthetic data)
- [ ] **REC-04**: UI shows active recording state, elapsed time, and an obvious stop control. — Phase 4 partial (real-time feedback needs manual)
- [ ] **REC-05**: Recording failure states identify permission, API, or output-file problems clearly. — Phase 4 partial (error cards implemented, real scenario unverified)
- [ ] **REC-06**: User can reveal or drag the recorded WAV for Cubase import. — Phase 4 partial (code paths exist, Cubase drag workflow needs manual)

### Downloader — Orphaned ⚠️

- [ ] **DL-01**: User can paste or enter a supported website URL for download. — Phase 5 orphaned (no VERIFICATION.md produced)
- [ ] **DL-02**: App detects yt-dlp availability/version and guides the user if it is missing or outdated. — Phase 5 orphaned
- [ ] **DL-03**: User can start a download job with output location and basic media/output options. — Phase 5 orphaned
- [ ] **DL-04**: Download jobs show progress, logs, completion, and errors. — Phase 5 orphaned
- [ ] **DL-05**: Downloaded files are added to the shared output inbox with source URL metadata. — Phase 5 orphaned
- [ ] **DL-06**: Downloader passes URLs and arguments safely without shell string interpolation. — Phase 5 orphaned
- [ ] **DL-07**: Downloader UI clearly scopes use to material the user is allowed to access and save. — Phase 5 orphaned

### Polish — Orphaned ⚠️

- [x] **UX-01**: User can reveal any output item in Finder. — Phase 3 validated
- [x] **UX-02**: User can drag generated files from the app or output inbox into Cubase/Finder-compatible targets. — Phase 3 validated
- [ ] **UX-03**: App provides concise, recoverable error messages for helper tools, permissions, and unsupported files. — Phase 6 orphaned (no VERIFICATION.md produced)
- [ ] **UX-04**: App provides keyboard shortcuts for frequent actions such as tap, reset, and start/stop recording. — Phase 6 orphaned (no VERIFICATION.md produced)
