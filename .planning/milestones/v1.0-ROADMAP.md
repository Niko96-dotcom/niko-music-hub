# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-11
**Phases:** 1-6
**Total Plans:** 19

## Overview

Outside Cubase Hub v1.0 delivers a native macOS tool hub with four production tools: BPM tapper, audio converter, internal audio recorder, and downloader hub. The architecture establishes a feature registry so new tools can be added without editing existing code. Core audio integration uses synthetic data pending real macOS 14.2+ Core Audio process tap verification.

## Milestone Summary

**Key Decisions:**

- Use SwiftPM as the first native macOS project structure so AppCore contracts are testable immediately.
- Keep Phase 1 navigation registry-driven and show only the registered Dev Tool.
- Tempo estimation remains pure Swift and Foundation-only so SwiftUI can consume deterministic state.
- Default conversion uses AudioPreset (44.1 kHz, 24-bit, preserve mono/stereo) — matches Cubase-ready WAV defaults.
- Native conversion writes `.tmp.wav` files and moves them only after WAVOutputVerifier passes.
- WAV metadata verification is reusable for conversion and future recorder outputs.
- External helper execution is centralized behind ExternalProcessRunning using Process.executableURL and argument arrays.
- FFmpeg fallback writes temporary WAV output and returns only after WAVOutputVerifier accepts the file.
- The WAV Converter registers as a normal ToolFeature with id wav-converter and capabilities producesFiles/runsJobs.
- Reveal and drag readiness are both gated by OutputHandoff, which only accepts .available existing .wav files.

**Issues Resolved:**

- Swift 6 sendability for WAVOutputVerifier file checks (stored FileManager replaced with @Sendable closure).
- AVAudioConverter input block Swift sendability warnings isolated in unchecked-Sendable helper.
- Swift required constants used by public initializer default arguments to be public.
- SPM project has no Info.plist (permission strings handled via AVCaptureDevice API).
- AsyncStream needed [weak self] capture to avoid retain cycle.
- AVAudioFormat sampleRate parameter is Double not Float.

**Issues Deferred:**

- REC-01-06: Real Core Audio process tap verification pending on macOS 14.2+ system.
- DL-01-07: Phase 5 verification never produced (no VERIFICATION.md).
- UX-03, UX-04: Phase 6 verification never produced (no VERIFICATION.md).
- Phase 3 human smoke checks (real app conversion flow, Finder/Cubase drag, live output inbox refresh) pending.

**Technical Debt Incurred:**

- CoreAudioTapAdapter generates synthetic audio (random noise), not real system audio.
- Phase 5 and Phase 6 completed without running execute-phase verification step.
- OutputInboxInspectorView uses onAppear-only refresh (new items may not appear until view recreated).
- 10 manual verification items for Phase 4 remain unexecuted.
