Niko Music Hub

A native macOS SwiftUI hub for Cubase archive recall and outside-Cubase production chores. Scan local archives read-only, find songs fast, preview the right mixdown, and keep generated files in one inbox.

View on GitHub Build locally
macOS 14.2+ Swift 6.x SwiftUI + AppKit Local-first Read-only archives

One app for the loop around Cubase.

Niko Music Hub keeps recall, conversion, recording, downloads, and output handoff in a single native workspace.

Archive Browser

Scan selected Cubase roots into songs, search across project and preview names, inspect diagnostics, and open the latest `.cpr`.

Preview Ranking

Pick the most useful mixdown with explainable role, folder, filename, version, extension, duration, and recency signals.

Output Inbox

Collect converted WAVs, recordings, and downloaded media in one app-owned handoff surface.

BPM Tapper

Tap tempo, switch half-time or double-time, save recent tempos, and copy values for session notes.

WAV Converter

Convert selected audio into Cubase-ready WAV presets with output verification and optional FFmpeg fallback.

Recorder And Downloader

Capture system audio where macOS supports it, download supported URLs with `yt-dlp`, and hand results to the inbox.

Built around archive safety.

The app is a recall layer, not a file manager. Source archives remain under user control.

Surface Allowed Blocked by design
Archive roots Enumerate folders, read metadata, read audio headers, open/reveal selected files. Rename, move, delete, rewrite, or create files inside the archive.
Generated outputs Write converter, recorder, and downloader outputs to the selected output folder. Writing generated files back into music archive roots.
Automation Fixture-based smoke tests and dry-run CPR open. Tests that require private archives or hidden local data.

Build it locally.

The repository is a Swift package. Local macOS gates are the source of truth.

git clone https://github.com/Niko96-dotcom/niko-music-hub.git
cd niko-music-hub
./script/build_and_run.sh

./script/ci.sh
./script/e2e_user_smoke.sh

The public fixture archive is generated from placeholders and test audio. No private Cubase projects or music files are needed to run the test suite.
