# Archive Model

Niko Music Hub treats a Cubase archive as a set of song folders selected by the user.

## Core Rules

| Rule | Detail |
| --- | --- |
| One child folder is one song | Each immediate child of a selected archive root is scanned as a song folder. |
| CPR files are versions | Every `.cpr` found under a song folder is listed as a project version. |
| Latest means modified time | The default CPR target is the `.cpr` with the newest modification date. |
| Previews are candidates | Supported audio files under a song folder become preview candidates. |
| Main preview is ranked | The ranker chooses the best candidate from deterministic signals. |
| Warnings are non-fatal | Missing CPRs, skipped root files, and invalid roots become diagnostics. |
| Archive writes are denied | The app does not create, edit, rename, move, or delete files inside archive roots. |

## Preview Candidate Signals

| Signal | Examples |
| --- | --- |
| Role | Full mix, master, instrumental, acapella, stem-like, unknown. |
| Folder | `Mixdown` is preferred over root, stems, and other folders. |
| Filename | Mix/bounce/master tokens help; stem/temp/reference tokens hurt. |
| Version | Higher parsed version wins when scores are otherwise equal. |
| Extension | WAV is preferred, then FLAC/AIFF/M4A/MP3 as tiebreaks. |
| Duration | Plausible song length helps; very short clips are called out. |
| Recency | Newer files win when earlier factors tie. |

## Searchable Fields

| Field | Example |
| --- | --- |
| Song display title | `Neon Hook` |
| Original folder name | `Preview Ranking Lab` |
| CPR filenames | `Neon Hook v2.cpr` |
| Preview filenames | `Lab Song v3 mix.wav` |
| Scan warnings | `No CPR project files found` |
| Skipped root entries | `LOOSE_FILE.txt` |

## Fixture Archive

The public fixture archive is generated by:

```bash
./script/fixtures/generate_cubase_archive_fixtures.sh
```

Fixture `.cpr` files are empty placeholders. Fixture audio is generated test media and does not contain private music.
