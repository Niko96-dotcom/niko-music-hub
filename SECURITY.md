# Security Policy

## Supported Version

| Version | Supported |
| --- | --- |
| v0.1.x | Yes |

## Reporting

Open a private security advisory on GitHub if available, or open an issue with minimal reproduction details and no private archive files attached.

## Data Safety Expectations

- Do not attach real Cubase projects or private music files to public issues.
- Prefer generated fixtures or small synthetic files.
- Report any behavior that writes to, deletes from, renames, or moves files inside a selected archive root as high priority.
